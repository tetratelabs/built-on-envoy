# <CLIName/> run

Run Envoy with extensions.

## Usage

<CodeBlock code="{{CLI_NAME}} run [options]" />

## Options

### `--extension <name>`

Extensions to enable by name. Can be specified multiple times or comma-separated.

<CodeBlock code="{{CLI_NAME}} run --extension rate-limiter --extension cors" />

### `--local <path>`

Path to a directory containing a local extension to enable. The directory must contain a `manifest.yaml` file. Can be specified multiple times or comma-separated.

<CodeBlock code="{{CLI_NAME}} run --local ./my-extension" />

### `--envoy-version <version>`

Envoy version to use (e.g., `1.31.0`). Can also be set via the `ENVOY_VERSION` environment variable.

<CodeBlock code="{{CLI_NAME}} run --extension rate-limiter --envoy-version 1.31.0" />

### `--log-level <component:level>`

Envoy component log level in the format `component:level`. Multiple components can be specified comma-separated. Use `all` to set the base log level.

| | |
|---|---|
| **Default** | `all:error` |

<CodeBlock code="{{CLI_NAME}} run --extension rate-limiter --log-level all:info,lua:debug" />

### `--listen-port <port>`

Port for Envoy listener to accept incoming traffic.

| | |
|---|---|
| **Default** | `10000` |

<CodeBlock code="{{CLI_NAME}} run --extension rate-limiter --listen-port 8080" />

### `--admin-port <port>`

Port for Envoy admin interface.

| | |
|---|---|
| **Default** | `9901` |

<CodeBlock code="{{CLI_NAME}} run --extension rate-limiter --admin-port 9000" />

### `--run-id <id>`

Run identifier for this invocation. Defaults to a timestamp-based ID (`YYYYMMDD_HHMMSS_UUU`). Use `0` for Docker/Kubernetes environments. Can also be set via the `BOE_RUN_ID` environment variable.

<CodeBlock code="{{CLI_NAME}} run --extension rate-limiter --run-id 0" />

## Global Options

These options apply to all commands:

### `--config-home <path>`

Configuration files directory. Can also be set via `BOE_CONFIG_HOME`.

| | |
|---|---|
| **Default** | `~/.config/boe` |

### `--data-home <path>`

Downloaded Envoy binaries directory. Can also be set via `BOE_DATA_HOME`.

| | |
|---|---|
| **Default** | `~/.local/share/boe` |

### `--state-home <path>`

Persistent state and logs directory. Can also be set via `BOE_STATE_HOME`.

| | |
|---|---|
| **Default** | `~/.local/state/boe` |

### `--runtime-dir <path>`

Ephemeral runtime files directory. Can also be set via `BOE_RUNTIME_DIR`.

| | |
|---|---|
| **Default** | `/tmp/boe-$UID` |

## Examples

### Run with a marketplace extension

<CodeBlock code="{{CLI_NAME}} run --extension rate-limiter" />

### Run with multiple extensions

<CodeBlock code="{{CLI_NAME}} run --extension rate-limiter --extension cors" />

### Run a local extension

<CodeBlock code="{{CLI_NAME}} run --local ./my-extension" />

### Run with custom ports

<CodeBlock code="{{CLI_NAME}} run --extension rate-limiter --listen-port 8080 --admin-port 9000" />

### Run with debug logging

<CodeBlock code="{{CLI_NAME}} run --extension rate-limiter --log-level all:debug" />

### Run with specific Envoy version

<CodeBlock code="{{CLI_NAME}} run --extension rate-limiter --envoy-version 1.31.0" />
