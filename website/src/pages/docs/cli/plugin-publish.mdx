# ee plugin publish

Publish your extension to the marketplace. Opens a pull request to the community repository.

## Usage

```bash
ee plugin publish <path> [options]
```

## Arguments

### `<path>`
Path to extension directory (default: current directory)

## Options

### `--dry-run`
Validate without publishing

### `--fork <repo>`
Use a specific fork for the PR

Example: `--fork myuser/envoy-ecosystem`

### `--branch <name>`
Branch name for the PR (default: auto-generated)

Example: `--branch add-my-filter`

### `--message <text>`
Custom PR description

## Examples

### Publish current directory
```bash
ee plugin publish
```

### Publish specific directory
```bash
ee plugin publish ./my-filter
```

### Dry run validation
```bash
ee plugin publish --dry-run
```

### Custom PR message
```bash
ee plugin publish --message "Add advanced caching filter with LRU eviction"
```

## Validation

The CLI validates:
- ✓ Manifest format and required fields
- ✓ Code compiles successfully
- ✓ Tests pass
- ✓ Documentation exists
- ✓ License is compatible
- ✓ No security vulnerabilities
