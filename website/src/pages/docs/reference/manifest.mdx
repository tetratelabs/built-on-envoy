# Extension Manifest

Extensions use a `manifest.yaml` file for metadata and configuration.

## File Location

```
my-extension/
└── manifest.yaml
```

## Complete Example

```yaml
name: my-filter
version: 1.0.0
type: lua
categories:
  - Security
author: Your Name
featured: false
description: Short description of the extension

longDescription: |
  Detailed description of the extension.
  Supports multiple paragraphs and markdown formatting.

tags:
  - security
  - authentication
  - jwt

license: Apache-2.0

examples:
  - title: Basic usage
    description: Simple configuration
    code: |
      ee run --plugin my-filter --config timeout=60s

  - title: Advanced usage
    description: With multiple options
    code: |
      ee run --plugin my-filter --config timeout=60s,enabled=true
```

## Field Reference

### `name`

| | |
|---|---|
| **Type** | `string` |
| **Required** | Yes |
| **Pattern** | `^[a-z][a-z0-9-]*$` |

Unique identifier for the extension. Must start with a lowercase letter and contain only lowercase letters, numbers, and hyphens.

```yaml
name: my-filter
```

### `version`

| | |
|---|---|
| **Type** | `string` |
| **Required** | Yes |
| **Pattern** | `^\d+\.\d+\.\d+$` |

Semantic version of the extension.

```yaml
version: 1.0.0
```

### `type`

| | |
|---|---|
| **Type** | `string` |
| **Required** | Yes |
| **Allowed values** | `lua`, `wasm`, `dynamic_module`, `composer` |

Type of the extension.

```yaml
type: lua
```

### `categories`

| | |
|---|---|
| **Type** | `array` of `string` |
| **Required** | Yes |
| **Allowed values** | `AI`, `Network`, `Observability`, `Performance`, `Transform`, `Security` |

Functional categories of the extension. An extension can belong to multiple categories.

```yaml
categories:
  - Security
  - Network
```

### `author`

| | |
|---|---|
| **Type** | `string` |
| **Required** | Yes |

Author or organization name.

```yaml
author: Your Name
```

### `description`

| | |
|---|---|
| **Type** | `string` |
| **Required** | Yes |
| **Max length** | 200 characters |

Short single-line description of the extension.

```yaml
description: JWT authentication and validation with JWKS endpoint support
```

### `longDescription`

| | |
|---|---|
| **Type** | `string` |
| **Required** | Yes |

Detailed multi-line description with markdown support.

```yaml
longDescription: |
  A comprehensive JWT authentication filter that validates
  JSON Web Tokens against configured issuers and audiences.

  Supports JWKS endpoints for key rotation and caching.
```

### `tags`

| | |
|---|---|
| **Type** | `array` of `string` |
| **Required** | Yes |
| **Min items** | 1 |
| **Pattern** | `^[a-z][a-z0-9-]*$` |

Searchable tags for categorization. Tags must be unique, start with a lowercase letter, and contain only lowercase letters, numbers, and hyphens.

```yaml
tags:
  - security
  - authentication
  - jwt
```

### `license`

| | |
|---|---|
| **Type** | `string` |
| **Required** | Yes |

SPDX license identifier.

```yaml
license: Apache-2.0
```

### `examples`

| | |
|---|---|
| **Type** | `array` of `object` |
| **Required** | Yes |
| **Min items** | 1 |

Usage examples for the extension. Each example must have `title`, `description`, and `code` fields.

```yaml
examples:
  - title: Basic usage
    description: Simple configuration
    code: |
      ee run --plugin my-filter --config timeout=60s
```

#### Example Object

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `title` | `string` | Yes | Short title for the example |
| `description` | `string` | Yes | Explanation of what the example demonstrates |
| `code` | `string` | Yes | Code snippet showing usage |

### `featured`

| | |
|---|---|
| **Type** | `boolean` |
| **Required** | No |
| **Default** | `false` |

Whether to feature this extension in UI/listings.

```yaml
featured: true
```

## Validation

Validate your manifest before publishing:

```bash
ee plugin publish --dry-run
```
